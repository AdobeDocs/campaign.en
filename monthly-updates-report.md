## Report: February 1, 2026 - February 24, 2026

### Updated Pages

- [Campaign security best practices](help/v8/config/security.md) - Feb 19 — FRR-RSC-05, FRR-RSC-08, FRR-RSC-09. Added Managed Cloud Services scope note and security checklist at top. New sections: Data and PII protection (HTTPS, PII restriction, passwords, sensitive pages), Authentication and session (Adobe IMS, session timeout), Instance and network security (IP allowlist, URL permissions, GPG keys), Operational and compliance (baseline comparison, audit trail). Expanded Access management (security groups, admin limits, product profiles) and Coding guidelines (no hardcoded secrets, input validation, SQL allow list). FRR-RSC-08: Statement that Campaign does not provide machine-readable guidance; link to Campaign Settings FAQ for comparing settings to recommended defaults.
- [Campaign Enhanced Security add-on](help/v8/config/enhanced-security.md) - Feb 19 — FRR-RSC-09. Description set to "Publicly available secure configuration guidance for the Campaign Enhanced security add-on". Added opening sentence that this page is part of Adobe's publicly available recommended secure configuration guidance for Campaign v8, with link back to security.md#public-guidance.
- [Frequently Asked Questions](help/v8/start/campaign-faq-comprehensive.md) - Feb 19 — FRR-RSC-05. New FAQ in Campaign Settings: "How do I check that my operators and permissions follow security best practices?" Describes where to view current settings (Administration > Access management, Control Panel), the recommended secure baseline (Security guidelines + Enhanced security add-on), and how administrators compare configuration to documented recommendations. Links to Get started with permissions, Security guidelines, Enhanced security add-on.
